Machine Learning Project Report

1. Motivation

This project addresses the challenge of applying various machine learning techniques to real-world regression and classification problems using datasets from the UCI Machine Learning Repository. We evaluate the performance of different model families including non-linear models (polynomials, step functions, splines, local regression, and GAMs), tree-based methods (bagging, random forests, and boosting), and Support Vector Machines with multiple kernels.

The goal is to systematically compare these approaches and identify which methods perform best for different types of prediction tasks, providing insights into model selection and evaluation in practical machine learning scenarios.

2. Datasets

2.1 Superconductivity Dataset (Regression Task)

Property Value
Source UCI Machine Learning Repository (ID: 464)
Task Type Regression
Samples 21,263 superconducting materials
Features 81 (all numeric)
Target Variable Critical temperature (Kelvin)

Description: The superconductivity dataset contains information about various superconducting materials and their critical temperatures - the temperature below which a material exhibits zero electrical resistance. The 81 features include chemical properties such as number of elements, atomic mass, atomic radius, density, electron affinity, fusion heat, thermal conductivity, and valence electrons, along with statistical measures (mean, weighted mean, geometric mean, entropy) computed across the constituent elements.

Significance: Predicting superconductor properties is crucial for advancing technology in power transmission, magnetic levitation, and quantum computing. Finding materials with higher critical temperatures could enable more practical applications without extreme cooling requirements.

2.2 Adult Census Income Dataset (Classification Task)

Property Value
Source UCI Machine Learning Repository (ID: 2)
Task Type Binary Classification
Samples 47,621 individuals (after cleaning)
Features 14 (numeric and categorical)
Target Variable Income level: ≤$50K (0) or >$50K (1)

Description: The Adult Census Income dataset, extracted from the 1994 U.S. Census database, contains demographic and employment information used to predict whether an individual earns more than $50,000 per year. The 14 features include:

  • Demographic: age, race, sex, native-country
  • Education: education level (categorical), education-num (years)
  • Employment: workclass, occupation, hours-per-week
  • Family: marital-status, relationship
  • Financial: capital-gain, capital-loss, fnlwgt (census sampling weight)

Significance: Income prediction is valuable for economic policy analysis, understanding factors that influence earning potential, and identifying income inequality patterns. This classic dataset has been widely used in machine learning research and serves as a benchmark for classification algorithms.

3. Method

3.1 Non-Linear Models

We implemented several non-linear transformation techniques:

  • Polynomial Features: Extended feature space with polynomial terms up to degree 3
  • Step Functions: Discretized features using quantile-based binning
  • Splines: Used B-spline transformations with varying knot counts
  • Local Regression: Applied LOESS/LOWESS for local smoothing
  • Generalized Additive Models (GAMs): Modeled as additive combinations of smooth functions

3.2 Tree-Based Methods

We evaluated ensemble tree-based approaches:

  • Bagging: Bootstrap aggregation with decision trees
  • Random Forests: Bagging with random feature subsets at each split
  • Gradient Boosting: Sequential ensemble with gradient-based optimization
  • AdaBoost: Adaptive boosting with weighted training instances

3.3 Support Vector Machines

We tested SVMs with four different kernel functions:

  • Linear Kernel: For linearly separable patterns
  • Polynomial Kernel: Degree-3 polynomial mapping
  • RBF Kernel: Radial basis function for non-linear patterns
  • Sigmoid Kernel: Neural network-inspired kernel

3.3 Baseline Comparison

We established baseline models using simple linear/logistic regression to provide reference performance metrics. All advanced models were compared against these baselines to quantify improvement and justify the added complexity.

3.4 Negative Results and Challenges

During experimentation, we encountered several challenges:

  • Some non-linear transformations (high-degree polynomials) led to overfitting
  • SVMs with certain kernels were computationally expensive on large datasets
  • Local regression methods showed limited scalability
  • Model performance varied significantly across different feature scales

5. Error Analysis

We conducted comprehensive error analysis including:

  • Residual Analysis: Examined prediction errors for systematic patterns
  • Confusion Matrices: Analyzed classification errors by class
  • Feature Importance: Identified key predictive features from tree-based models
  • Cross-validation: Ensured model generalization across data splits
